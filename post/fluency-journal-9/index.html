Working in Process- Character Model Implantation

In terms of the issues two last week, I start to learn how to implement the walking animation into Unity Game system and add the control function. Before choosing to use Blender, I heard that the version of Unity could support Blender, then I can see directly model and animation in the later development process without exporting fbx. This led to the successful animation of the characters walking in the first few weeks, and thought that the fdx file could be exported directly. However, it turns out that in Unity, you cannot use fdx and Blender files directly. So I still need to implement this animation effect by creating new functions and control commands in Unity. So I made some attempts to put the Blender file directly into the assets file of Unity, and Unity directly to read. However, then I quickly discovered a new problem. If I link or append other files in my Blender file, I will display any error messages in the free path, and Unity cannot directly recognize Blender's file reference path.

So I started a new round of bug fixes and new features. Fortunately, I found many solutions to achieve animation. Then I found that there are three main ways to implement the model for developers who already have a model.

The first way is to set up the model in the model editor such as Blender and edit the action of the model. This way, after importing Unity, such a model can be acted upon. This is also the way I use it now, but in this way, when it is applied, there are problems beyond my ability. I still cannot solve these bugs during the continuous debugging and testing of the code. The second way is to search for the motion action in Unity's asset store if you only have the model, and then take the action of the appropriate model and bring it into the animation control. However, this approach is risky. This approach should be banned from use in course projects, and many suitable plugins cost much money. The third way is to use a model editor such as Blender, create a human model, set up skeletal animation and then set the action of the model in Blender and then pour into Unity.

So I decided to try a third way to connect the blender file to the Unity. By integrating multiple animations, the state machine is used to play and switch animations. I use the Animator component to control the playback of character animations, and the Animator Controller is a file type that exists as a separate configuration file in Unity to provide profound control over animation playback through scripts. The two core components required are the Animator Controller that controls the animation playback logic and the animated skeleton Avatar object.

using UnityEngine;
using System.Collections;

public class TestAnimChange : MonoBehaviour
{
    private Animator _animator;

    void Start()
    {
        _animator = this.GetComponent<Animator>();
    }
    
    void Update()
    {
        if(Input.GetKeyDown(KeyCode.W))
        {
            _animator.SetBool("Walk", true);
        }
        if(Input.GetKeyUp(KeyCode.W))
        {
            _animator.SetBool("Walk", false);
        }
        if(Input.GetKeyDown(KeyCode.R))
        {
            _animator.SetBool("Run", true);
        }
        if(Input.GetKeyUp(KeyCode.R))
        {
            _animator.SetBool("Run", false);
        }
    }
}

The User Journey Map

With interaction paradigms and interaction patterns, I also need to be responsible for user journey maps and designs. For our demo, we decided to create a customer journey for the main character. Based on our target audience, we decided to use a Korean beginner who encountered difficulties as a user group image. He wants to learn Korean but finds it difficult to find a suitable way to learn and to continue learning. In the process, I noticed the process of starting to empathize with the user and understand the psychological changes of the user as a designer. It is indeed a valuable tool that helps us understand the needs of users, the feelings and expectations of users. Find the pain points and solutions of the design in the process of psychological change. I also noticed that when I completed each step of the process, I was able to identify gaps and features in the system. For example, at the beginning of the concept, we thought that a feature was applicable to the user, but in the actual experience, the user might not choose to use it at all.

Working in Process- Issues 3

In terms of the Walk animation design, I found the character always walking in a different direction，which always made me wanna cry and felt too tired.

After selecting the Walk animation in the Project panel, I can preview the Walk animation in the Inspector panel. However, I found that the character has an absolute offset in the X-axis direction during walking. At this point, I try to adjust the Offset offset so that the Average Velocity attribute has a zero amount of motion on the x-axis. I want to control the character model to run along the x-axis in this way to produce an angular offset.
